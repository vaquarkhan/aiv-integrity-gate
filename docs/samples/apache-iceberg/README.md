# Apache Iceberg AIV Design Rules

A comprehensive set of AIV rules for Apache Iceberg. These rules detect and reject AI slop (emoji, attribution markers, boilerplate comments), logic that does not match Iceberg standards, and code that violates the project architecture. Adapt the rules for your own project.

**Author:** Vaquar Khan

---

## What Gets Rejected

| Category | What Gets Rejected |
|----------|--------------------|
| Emoji in code or comments | Check marks, lock icons, rockets, and similar symbols |
| Attribution markers | Phrases like "Generated by" with tool names in comments |
| Boilerplate comments | Generic helper phrases like "I'll help you" in comments |
| Wrong snapshot logic | In-memory list manipulation instead of the ExpireSnapshots API |
| Non-existent or deprecated APIs | Calls to APIs that do not exist or are deprecated |
| Architecture violations | Direct schema mutation, missing `.commit()`, wrong transaction pattern, raw filter strings, direct partition or sort mutation |
| Standards violations | Missing Apache license, Serializable, Thread.sleep, Assert.fail |

## Rules Summary

### AI Slop Detection (Reject)

| Rule | Catches |
|------|---------|
| `no-emoji-in-code` | Emoji in code/comments (common AI slop tell) |
| `no-ai-generated-markers` | Attribution phrases like "Generated by ChatGPT" left in committed code |
| `no-ai-boilerplate-comments` | Generic helper phrases ("I'll help you", "As an AI") in comments |

### Snapshot Logic and API Correctness

| Rule | Forbids | Requires |
|------|---------|----------|
| `snapshot-expiration-must-use-api` | removeSnapshots, toRemove.add, MetadataUpdate.RemoveSnapshot | expireSnapshots |
| `expire-snapshots-must-commit` | — | .commit |
| `schema-must-use-update-schema` | schema.add, schema.remove, schema.rename, schema.set | — |
| `transaction-must-commit` | — | commitTransaction |
| `pending-update-must-commit` | — | .commit (when updateSchema, newAppend, replaceSortOrder, removeOrphanFiles, etc.) |

### Architecture: Evolution and Immutability

| Rule | Forbids | Requires |
|------|---------|----------|
| `add-column-must-use-types` | — | Types. (when addColumn) |
| `filter-must-use-expressions` | — | Expressions. (when .filter() used) |
| `no-direct-partition-spec-mutation` | partitionSpec().addField, partitionSpec().removeField | — |
| `no-direct-sort-order-mutation` | sortOrder().asc(, sortOrder().desc( | — |

### Deprecated APIs

| Rule | Forbids |
|------|---------|
| `no-deprecated-remove-snapshot` | MetadataUpdate.RemoveSnapshot |
| `no-deprecated-file-rewriter` | FileRewriter, SizeBasedFileRewriter, BinPackRewriteFilePlanner |
| `no-deprecated-system-properties` | SystemProperties |

### Tests and Production

| Rule | Forbids |
|------|---------|
| `no-thread-sleep-in-tests` | Thread.sleep |
| `no-assert-fail-in-tests` | Assert.fail( |
| `no-system-exit` | System.exit |
| `no-system-out` | System.out.println, System.err.println |
| `no-java-serialization` | implements Serializable |

### Standards

| Rule | Requires |
|------|----------|
| `apache-license-header` | "Licensed to the Apache Software Foundation" |
| `deprecation-requires-javadoc` | @deprecated when @Deprecated present |

## Why These Rules Matter

These rules catch common contributor mistakes that traditional CI often misses:

| Mistake | AIV Catches | Why It Matters |
|---------|-------------|-----------------|
| `scan.filter("col = 5")` | `filter-must-use-expressions` | Iceberg requires `Expressions.equal("col", 5)` — raw strings do not bind to the schema |
| `addColumn("x", Type.LongType.get())` | `add-column-must-use-types` | Must use `Types.LongType.get()` — wrong factory breaks evolution |
| `partitionSpec().addField(...)` | `no-direct-partition-spec-mutation` | Partition spec is immutable — use `updateSpec().addField().commit()` |
| `sortOrder().asc("id")` | `no-direct-sort-order-mutation` | Sort order is immutable — use `replaceSortOrder().asc().commit()` |
| `replaceSortOrder()` without `.commit()` | `pending-update-must-commit` | PendingUpdate must be committed — the change never applies otherwise |
| `removeOrphanFiles()` without `.commit()` | `pending-update-must-commit` | Same — orphan removal is a metadata operation |

## Value

AIV with these rules detects AI slop (emoji, attribution markers) and rejects code that does not match Iceberg architecture (wrong snapshot logic, wrong APIs, direct partition or sort mutation, raw filters without Expressions) — before they reach human reviewers.
