# Gate: `design`

## Why it exists

Enforces **project-specific rules** from `.aiv/design-rules.yaml`: forbidden APIs, required calls when certain keywords appear, and common **AI slop markers** (emoji, “Generated by …” strings) if you keep those constraints.

## When a finding is wrong

Substring rules can match **comments** or **strings** that mention a forbidden name. Narrow rules with `keywords`, or rephrase documentation.

## How to fix

- Remove or replace forbidden calls; add required calls where your policy demands them.
- Split oversized files so constraints apply only where intended.
